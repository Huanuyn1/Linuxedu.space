Linux 的家，
每个房间都有名字。

第一次面对 / 一脸懵很正常。记住这几个房间，找东西不再靠缘分。

路径是什么什么时候会去
~（即 /home/你）你的主目录，桌面、文档、下载都在这99% 的日常操作
~/.config各软件的配置文件调设置、折腾美化（Ricing）
/etc系统级配置改源、改系统设置（先备份再动手）
/var/log系统和应用日志出问题时来这里破案
/usr/bin安装好的命令本体好奇某个命令住在哪的时候
/opt第三方大软件的常规住处手动装某个闭源软件时
/tmp临时文件，重启即清空随手扔中间产物
一分钟看懂

权限数字是什么黑话？

每个文件有三组权限：所有者 / 所属组 / 其他人，每组由三种动作组成：

  • r（读）= 4，w（写）= 2，x（执行）= 1
  • 每组数字 = 三种动作相加，如 rwx = 4+2+1 = 7
  • 所以 chmod 755 的意思：我能读写执行，你们只能读和执行
照抄即可

三个背下来就够的数字

  • 644 —— 普通文件：我能改，别人能看。作业、文档、代码都用它
  • 755 —— 脚本和目录：所有人都能进、能执行
  • 600 —— 私密文件：只有我能读写。SSH 私钥必须是它，不然 ssh 拒绝工作

拿不准就先 ls -l 看看别人怎么设的，比瞎猜靠谱。

单招会了，
该学连招了。

命令行的威力在组合。下面四套连招覆盖日常 80% 的「救救我」场景，建议直接收藏。

磁盘满了？揪出元凶

从大到小列出当前目录的占用，第一名往往就是罪魁祸首（通常是某个没删的虚拟机镜像）。

du -sh * | sort -rh | head
# du 算大小 → sort 按人类可读数字倒序 → 只看前 10

程序卡死？找到并送它走

先礼后兵：kill 默认发的是「请你退出」；只有它装死时才用 kill -9「就地正法」。

ps aux | grep 程序名
kill 进程号
# 还不行才放大招：kill -9 进程号

服务起不来？边重启边看日志

journalctl 的 -f 和 tail -f 一样实时滚动，改配置、重启服务时盯着它，报错第一时间现形。

sudo systemctl restart 服务名
journalctl -u 服务名 -f
# 想退出实时日志：Ctrl + C

忘了上次怎么敲的？搜历史

上个月那条完美的 ffmpeg 命令不用重新研究，历史记录里都躺着呢。

history | grep ffmpeg
# 更快的手法：Ctrl + R，输入关键词即时搜
红牌区 · 敲回车前默读三遍

这几条命令，专治手滑和冲动

  • rm -rf —— 不询问、不进回收站、不可恢复。路径里出现 / 或 * 时，停下来再看一眼。删之前先 ls 确认站在哪个目录。
  • sudo —— 管理员权限意味着系统不拦你。网上抄的命令加 sudo 之前，先搞懂它干什么。
  • > 重定向 —— 覆盖是静默的。> 重要文件 手一滑，文件当场清零。想追加内容请用 >>。
  • dd —— 磁盘级操作，源和目标写反 = 把系统盘抹掉。人送外号 disk destroyer，烧录启动盘时务必用 lsblk 再三确认设备名。
  • curl ... | bash —— 一键脚本很方便，但等于把键盘交给陌生人。先下载下来读一遍再跑，至少确认网址是官方的。

真正的护身符只有一条：重要数据有备份。备份到位，上面这些全是纸老虎。
