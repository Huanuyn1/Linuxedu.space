🐣 入门概念 12 条

先把世界观的积木搭好，后面的一切都好懂。

发行版Distro

Linux 的「品牌套餐」：同一个内核，配上不同的软件、桌面和理念打包出售。Ubuntu、Arch、Debian 都是发行版——就像同样的发动机，装进了不同品牌的车。

内核Kernel

系统真正的核心，负责跟硬件对话。你平时说的「Linux」严格来说只是它，平时用的完整系统其实是「内核 + 一堆软件」的组合。

桌面环境DE

你看到的桌面、任务栏、设置中心的总和。GNOME 像 macOS，KDE 功能多到像瑞士军刀，Xfce 轻快适合老电脑。同一个发行版可以换着用。

包管理器Package Manager

Linux 的应用商店 + 自动更新二合一，不过是命令行版的。一行命令装软件，还自动处理依赖。各家的叫法：Ubuntu 用 apt，Arch 用 pacman，Fedora 用 dnf。

软件仓库 / 源Repo

发行版官方维护的软件大超市，包管理器从这里进货。说它「官方」是因为里面的软件都经过审核，比在网页上乱下安装包安全得多。

镜像站Mirror

软件仓库在国内的「分店」。默认源在国外下载慢，换成清华、中科大等镜像站，装软件速度起飞。换源是装完系统的第一课。

滚动更新Rolling Release

没有「大版本」，系统天天小步快跑，软件永远最新。优点是新鲜，代价是偶尔会「滚挂」——更新完出毛病。Arch 就是滚动派掌门。

LTS长期支持版

追求稳定的版本策略：一个版本官方维护 3～5 年，只修 bug 不追新。求稳的同学认准 LTS 三个字母，比如 Ubuntu 24.04 LTS。

Live USB试用 U 盘

把系统装到 U 盘里开机就跑，不碰硬盘。买电脑、装系统前的「试驾」神器：先在 Live 模式里试试 Wi-Fi、触板好不好使，再决定装不装。

基于 / 血统Based on

「Mint 基于 Ubuntu，Ubuntu 基于 Debian」——发行版也有族谱。基于谁，就继承谁的软件仓库和脾气，所以查教程时血统比名字更重要。

GNU/Linux

严格党的执念：完整系统是 GNU 工具集 + Linux 内核，所以自由软件基金会坚持要叫它 GNU/Linux。日常说 Linux 没人拦你，但在老帖子里看到这个写法别懵。

开箱即用

装完不做任何配置就能直接干活。Mint、Ubuntu 走这条路；Arch 开箱只有一行命令行——它管这个叫自由。

⌨️ 命令行黑话 12 条

黑窗口里的黑话。别怕，它们大多比听起来善良。

终端 / ShellTerminal / Bash

终端是那个黑窗口，Shell 是里面听你指挥的「管家」（Bash、Zsh 都是 Shell 的品种）。日常混用没问题，别在考试里较真。

sudo

「以管理员身份执行」。相当于 Windows 的右键管理员运行。经典梗：说不通的人，在前面加个 sudo 就通了——sudo 帮我带饭。

root

系统里的超级管理员，权力大无边，也能一句话把系统扬了。平时用普通用户 + sudo 就好，别图省事常驻 root。

家目录~

你的个人文件夹，相当于 Windows 的「用户目录」，用 ~ 表示。你的文档、配置、黑历史都在这。备份先备它。

管道|

把前一个命令的输出塞给后一个命令当输入，像流水线接力。例：ps aux | grep qq ——先列出全部进程，再筛出带 qq 的行。

重定向> 和 >>

把命令的输出写进文件：> 是覆盖写，>> 是追加写。例：echo "我爱 Linux" > 日记.txt。

守护进程Daemon

在后台默默干活的程序，比如 Web 服务、打印服务。名字听着吓人，其实是因为英文 daemon 原意是「守护精灵」，不是恶魔。

chmod 777

给文件「所有人都能读写执行」的权限。社区著名反面教材：出问题就 777 是莽夫行为，正确做法是搞清楚谁需要什么权限。

TTY

纯命令行的全屏终端。图形界面卡死时按 Ctrl+Alt+F3 能进 TTY 救场，按 Ctrl+Alt+F2（或 F1）切回图形界面。记住它，关键时刻救命。

rm -rf

删库跑路的凶器：递归、强制、不给确认地删除。社区头号恐怖故事主角。看到任何带 -rf 的命令先读三遍再回车，来历不明的一律不执行。

man 手册

man 加命令名 = 查官方说明书，比如 man ls。退出按 q——新人第一次进去都以为电脑卡死了。嫌太学术就装个 tldr，直接给你人话版例子。

环境变量 / PATH

系统级的全局小纸条。PATH 决定你敲一个命令时系统去哪儿找它——遇到 command not found，八成是程序的路径没加进 PATH。

📦 装软件黑话 8 条

装个软件而已，怎么有这么多门派。

Flatpak

跨发行版的通用打包格式，自带运行环境，哪个发行版都能装，软件商店叫 Flathub。代价是体积大一点——沙箱里装了它的半个世界。

Snap

Ubuntu 力推的通用包格式，沙箱隔离、自动更新。口碑两极：方便是真方便，启动偏慢和强制自动更新也是真有人烦。

AppImage

下载即用的「绿色软件」：一个文件双击就跑，不安装、不污染系统，删了就干净。适合尝鲜和装进 U 盘随身带。

.deb / .rpm

两种安装包格式：deb 给 Debian/Ubuntu 系，rpm 给 Fedora/openSUSE 系。相当于安装包界的两大流派，下载时别进错家门。

依赖 / 依赖地狱

软件运行要借别人的库，这些「别人」就是依赖。如今包管理器自动搞定一切；早年要手工一个个装，装到怀疑人生，史称依赖地狱。

编译安装

从源码自己构建软件，经典三连：./configure、make、sudo make install。现在大多数时候用不上，但 Gentoo 用户把它过成了日常。

AUR Helper

AUR 的装机助手，yay 和 paru 最流行，一条命令自动下载、编译、安装 AUR 包。官方态度：用可以，先学会自己看一眼 PKGBUILD。

源外安装

从官网、GitHub 直接下载安装包。能装，但失去了仓库的审核和自动更新。建议顺序：官方源 → Flatpak → 官网，别一上来就当下载站战士。

🔧 装机与硬件 12 条

从按下开机键到点亮桌面，路上会撞见的词。

UEFI / BIOS

开机最先跑的固件程序，负责把系统叫醒。UEFI 是 BIOS 的现代版。装系统、关 Secure Boot 都在它的设置界面里操作（开机猛按 F2/Del/F12）。

Secure Boot安全启动

UEFI 的一项安全检查，有时会拦住 Linux 启动盘。装系统卡在这一步就去 BIOS 里把它关掉——这是新手装机最高频的坎。

GRUB引导器

开机时的系统选择菜单，负责把内核叫醒。双系统靠它选边站；它罢工的表现是开机黑脸，别慌，一根启动盘就能修好。

EFI 分区ESP

存放引导文件的小分区，几百 MB 就够，双系统里 Windows 和 Linux 共用同一个。手滑格式化它 = 两个系统一起进不了门。

分区 / 挂载

把硬盘切成几块用（系统、数据分开），「挂载」就是把某块分区接到目录树的某个位置。双系统就是给两个系统各分一块地盘。

文件系统ext4 / Btrfs

硬盘上组织文件的方式。ext4 稳如老狗，是默认之选；Btrfs 自带快照功能，是后悔药 Timeshift 的好搭档。装系统时默认选项就够用。

双系统Dual Boot

一台电脑装两个系统，开机时三秒选一个。Windows 当游戏厅，Linux 当书房，成年人不做选择。新手上路最稳妥的姿势。

核显 / 独显

核显长在 CPU 里，省电够用，Linux 下驱动开源免折腾；独显是独立显卡，性能强适合游戏和炼丹，NVIDIA 的驱动是闭源的（但现在一键能装）。

开源 / 闭源驱动

Intel、AMD 显卡驱动是开源的，系统自带；NVIDIA 官方驱动闭源，要单独装。开源替代品 Nouveau 性能一般，打游戏还是用官方驱动。

swap交换分区

硬盘上划出一块当「应急内存」，内存不够时救场，也是休眠功能的存放点。内存 16G 以上给 4～8G 意思一下就行。

快照 / 后悔药Timeshift

给系统拍照片存档，折腾坏了十秒回滚到昨天。和「黑话」没关系，但每个老玩家都会拍着大腿劝你装一个。

变砖Brick

设备被折腾到开不了机，功能退化成一块砖。手机圈黑话，刷 BIOS、乱动引导时也会用上。应对：操作前查好教程，手边常备启动盘。

⚙️ 系统进阶词 8 条

日用不一定碰，看帖一定会遇到的进阶词汇。

systemd

系统的总管家：开机后第一个进程，负责拉起所有服务。社区对它爱恨两极，但你天天都在用——systemctl 命令就是它的遥控器。

Wayland / X11

两代显示服务器：X11 是服役三十多年的老将，Wayland 是更现代、更丝滑的接班人。新手默认 Wayland 就好，个别老软件水土不服时再查切换方法。

窗口管理器WM

不要桌面环境，只留一个管窗口的极简班子：i3、Sway、Hyprland。平铺式自动排窗，键盘流玩家的归宿，Ricing 重灾区。

内核恐慌Kernel Panic

内核遇到处理不了的错误，直接躺平罢工——相当于 Windows 蓝屏。稀有事件，真遇上了先拍照留念，再回想最近动了什么。

输入法框架fcitx

中文输入的地基：fcitx5 配拼音或 Rime 是主流方案。输入法闹脾气别砸键盘，九成是框架的环境变量没配对。

日志journal

系统的流水账，用 journalctl 查看。排查问题的第一现场：服务起不来？journalctl -u 服务名，看日志比瞎猜快十倍。

GCC编译器

Linux 世界的默认编译器套装。学过 C 语言的都跟它打过交道：gcc hello.c -o hello，以及随之而来的四行报错——那是青春的一部分。

容器 / Docker

轻量级的「打包带走」：把应用连环境一起装进集装箱，在哪台机器跑都一样。课程实验、部署服务的救星，从此告别「在我电脑上明明是好的」。

🏘️ 社区黑话 14 条

论坛、群聊、帖子里流通的硬通货。

AURArch 用户仓库

Arch 用户自发上传的软件仓库，十万软件包什么都有。是 Arch 最大的魅力，也要求你留个心眼——毕竟是网友上传的，装前瞄一眼评论。

PPA

Ubuntu 的第三方软件源，用来装官方仓库没有（或太旧）的软件。相当于给应用商店加了个「代购渠道」。

RTFM

「Read The Fine Manual」——去读手册！社区回答伸手党的经典回复。礼貌版说法是「这个在 Arch Wiki 上有哦」。提问前先搜索，是社区第一课。

Arch Wiki

社区公认的神级文档，写得比很多官方文档都好。就算你不用 Arch，查配置也常常查到它头上——用别的发行版看 Arch Wiki，是公开的秘密。

Distro Hopping

发行版跳跃症：不停地换系统尝鲜，Mint 用两周换 Fedora，再换 Arch……老玩家的必经之路。放心，大部分人会最终在某一站安家。

滚挂

滚动更新翻车的简称：一更新，系统出毛病了。应对：更新前看一眼公告，装好快照工具，滚挂了也能十秒回滚。

日用

把 Linux 当主力系统天天用。「这个发行版能不能日用」=「靠不靠谱，能不能托付四年」。

炼丹

训练 AI 模型的戏称：参数是药材，显卡是丹炉，loss 不降就是丹炸了。Linux + NVIDIA 是炼丹界标准道观。

Ricing

桌面美化装修：换主题、调毛玻璃、配状态栏，把桌面打扮得比 macOS 还好看。源自汽车圈的「爆改」，是 Linux 玩家的重要快乐源泉。

「BTW, I use Arch」

「顺便一提，我用 Arch」——调侃 Arch 用户逢人必提的梗。你在本站首页看到的有咲同学就是典型患者。

FOSS / 自由软件

Free and Open Source Software。自由软件的「自由」是自由的自由，不是免费——虽然通常也免费。GPL 协议还带「传染性」：用了我的代码，你的也得开源。

上游 / 下游Upstream

软件的原始作者是上游，打包改造的发行版是下游。「报 bug 要报给上游」= 找亲妈告状，别找错人。

用爱发电

开源开发者义务劳动的自嘲：不收钱，纯靠热爱维护。所以对维护者客气点——提 Issue 时把情况讲清楚，就是你交的最好电费。

企鹅 / Tux

Linux 吉祥物，一只憨憨的企鹅。自称「企鹅人」的，就是 Linux 用户的身份证。来源：Linus 本人被企鹅咬过（是真的）。

🎮 游戏与炼丹 8 条

打游戏和跑模型，大学生两大显卡消耗场景。

Wine

让 Windows 软件跑在 Linux 上的兼容层，名字是个递归梗：Wine Is Not an Emulator。能跑，但调校带点玄学，适合折腾党。

Proton

Valve 基于 Wine 魔改的游戏兼容层，Steam 里一键开启，几万款 Windows 游戏直接玩。Linux 游戏生态的第一功臣。

Proton-GE

社区增强版 Proton，补丁更激进、新游戏支持更快。官方版跑不动时的民间偏方，用 GE-Proton 管理器一键安装。

ProtonDB

玩家共建的游戏兼容性数据库，查一款游戏在 Linux 上的表现先看它：白金 = 开箱即玩，青铜 = 自求多福。

反作弊Anti-Cheat

网游的反外挂系统，部分厂商拒绝支持 Linux，是游戏党最大的拦路虎。单机随便玩，竞技网游先去 areweanticheatyet 查名单。

Steam Deck

Valve 的掌机，跑的是 Arch 底座的 SteamOS。它的存在本身，就是「Linux 能打游戏」这块广告牌本牌。

CUDA

NVIDIA 的 GPU 计算平台，炼丹的入场券。深度学习框架大多以它为准——这就是大家劝你炼丹选 N 卡的原因。

显存VRAM

显卡自带的内存，炼丹时的丹炉容量：模型越大吃得越多。6GB 入门、12GB 从容、24GB 为所欲为。
